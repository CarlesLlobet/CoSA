SQLmap Usage guide

What is it?

SQLmap is an open source tool to do peneteration tests, automating the detection and exploitment of SQL injection vulnerabilities and Database Servers control.

How to use it?

Name

This field allows us to identify the task
It is mandatory to introduce it to be able to differentiate our many tasks, even if they are pointing to the same target, but with different configurations.

Target

This field defines the hosts we want to scan over. We have two options, and can choose between them or use both:

  • Domain: Here we are capable of inserting domains so OpenVAS resolves them for us, separated by comma, and with the unique condition that they have to be complete to be able to found the servers.

  • IPs: Here we can insert IPs separated by comma of the hosts we want to scan directly. In case we want to add a whole network (e.g: 147.83.1.0/24), you will need a special permission from the administrator.

Parameters

  • Verbosity: Defines the verbosity level of our results, the amount of detail in the output. There are 7 levels:
    • 0: Just python error and critical messages.
    • 1: Also useful information and alert messages.
    • 2: Also debug messages.
    • 3: Also injected payloads.
    • 4: Also HTTP requests.
    • 5: Also HTTP response headers.
    • 6: Also the page content in the HTTP responses.

  • Level: Defines the amount of payloads it will try to inject, without being an exact amount. It will also affect to the typology of tests to execute. With level 1 just GET and POST parameters will be tested.Defineix la quantitat de payloads que intentarà injectar, sense ser el From level 2 also values of HTTP cookies, and from level 3 HTTP headers User-Agent/Refearer.

  • Risk: Defines the risk the tool will accept when testing the database. There are three levels:
    • 1: Innocuous to the great majority of injection points.
    • 2: It could cause a DDOS to the Database.
    • 3: It could even DELETE data from the database.

  • Depth: SQLmap also allows crawling a URL, and maybe the domain we are testing has no vulnerability, but one closer does. This parameter allows to define the depth level of search that SQLmap will test, being 1 just the given domain, and 10 even links at 10 clics from the given URL.

  • Charset: Charset of the database to analyse. UTF-8 is the default.

  • Email notification: Here we can check if we want to receive an email when the task is finished.

  • Finish: Finally we can choose if we just want to save the task, or execute it after saving.
