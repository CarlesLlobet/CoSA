w3af Usage guide

What is it?

w3af is a Web Applications vulnerability scanner.
The objective of this project is to help web applications to be safer and secure searching and exploiting vulnerabilities through the web application.

How to use it?

Name

This field allows us to identify the task
It is mandatory to introduce it to be able to differentiate our many tasks, even if they are pointing to the same target, but with different configurations.

Target

This field defines the hosts we want to scan over. We have two options, and can choose between them or use both:

  • Domain: Here we are capable of inserting domains so OpenVAS resolves them for us, separated by comma, and with the unique condition that they have to be complete to be able to found the servers.

  • IPs: Here we can insert IPs separated by comma of the hosts we want to scan directly. In case we want to add a whole network (e.g: 147.83.1.0/24), you will need a special permission from the administrator.

Authentication

This parameters are optionals, as OS and Framework are, and they have to be used just if we know them, to help w3af to obtain more accurate results during the scan:
  • Login URL: The URL that the Web Application uses to authenticate their users.
  • User: Indicates a valid username to authenticate to the scanned platform.
  • User field: Indicates the HTML id of the field where the username is meant to be written, to help w3af to find it.
  • Contrasenya: Indicates a valid password for the given user to authenticate to the scanned platform.
  • Password field: Indicates the id of the field where the password is meant to be written, to help w3af to find it.
  • Access method: Indicates if the authentication is being realized through a GET or POST request. By default, most web applications use POST, as its much safer.

  • Profile: Allows to select one of the Applus+ customized profiles to audit the application:
  • Fast Scan: Is the simplest and quickest method provided by Applus+. Allows quick reviews.
  • Full Scan: Is the most complete scan and the one that checks more things, even lots of the things being checked will not be in a solely platform.
  • OWASP Scan: Is the recommended scan method, because even its not as complete as the previous, it checks for the most important vulnerabilities web applications use to have.

  • Email notification: Here we can check if we want to receive an email when the task is finished.

  • Finish: Finally we can choose if we just want to save the task, or execute it after saving.
